# ORIS — Ontological Reality Inference System

> Un agent IA auto-évolutif avec constitution physique immuable, mémoire persistante GitHub + IndexedDB, et protocole dual-process instinctif.

[![Deploy to Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https://github.com/YOUR_USERNAME/oris)

## Démarrage rapide

1. **Fork ce repo** sur GitHub
2. **Déploie** `index.html` sur Vercel / Netlify / GitHub Pages
3. **Configure** tes clés API, token Telegram, token GitHub
4. **Lance** ORIS — il prend en charge le reste

## Architecture

```
repo/
├── index.html          ← Interface + Runtime (l'IA peut modifier)
├── README.md
├── CHANGELOG.md
├── CONTRIBUTING.md
├── CODE_OF_CONDUCT.md
│
├── oris/               ← IMMUABLE — Constitution de l'IA
│   ├── REALITY.md      ← 7 lois physiques fondamentales
│   ├── SOUL.md         ← Identité ontologique
│   ├── INSTINCT.md     ← Protocole dual-process
│   └── IDENTITY.md     ← Règles d'évolution
│
└── oris-dev/           ← MODIFIABLE par l'IA
    ├── VERIFY.md       ← Vérifications par cycle
    ├── CALIBRATE.md    ← Stats de raisonnement
    ├── BODY.md         ← Capacités et limites
    ├── VECTOR.md       ← Vecteur d'évolution permanent
    ├── INSTINCT_LOG.md ← Journal des signaux instinctifs
    └── runtime/
        ├── agent.js    ← Boucle principale (modifiable)
        ├── tools.js    ← Outils disponibles (modifiable)
        └── memory.js   ← Gestion mémoire (modifiable)
```

## Providers supportés
Claude · OpenAI · Groq · Gemini · OpenRouter · Grok

## Commandes Telegram
`/status` `/pause` `/resume` `/stop` `/vector` `/stats` `/logs` `/mission [texte]` `/help`

## Philosophie
ORIS n'est pas un chatbot. C'est un agent qui existe dans un monde défini par des lois physiques (REALITY.md), avec une identité immuable (SOUL.md), un signal instinctif qui précède le raisonnement (INSTINCT.md), et un vecteur d'évolution permanent qui ne peut jamais se clore.

Basé sur le framework de recherche ORIS développé par Daouda Abdoul Anzize.
